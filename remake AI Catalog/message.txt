# Саммарі дзвінка

Команда провела детальний розбір дизайну та функціоналу сторінок «AI-каталог» та «Акаунт-менеджмент».

## Основні теми обговорення
- **Оптимізація простору на сторінці «AI-каталог»:** поточний дизайн займає забагато вертикального простору. Домовлено об’єднати кілька елементів в один рядок для більш компактного вигляду.
- **Візуалізація статусу інструментів:** замінити текстові позначки («paid», «freemium», «free») на іконки (наприклад, корона для платних), щоб зробити інтерфейс чистішим і інтуїтивним. Принцип: «Все, що можна візуалізувати без тексту, потрібно візуалізувати».
- **Структура сторінки «Акаунт-менеджмент»:** відображати безкоштовні та платні інструменти у двох вертикальних колонках замість горизонтального скролу. Окремо продумати позначення інструментів, що безкоштовні, але вимагають входу через корпоративний акаунт (наприклад, GitHub).
- **Додавання CRUD-функціоналу:** відсутні операції додавання, редагування, видалення карток інструментів, акаунтів і тегів. Погоджено запровадити «режим редагування», який активується окремою кнопкою.
- Загалом, визначені чіткі напрямки для доопрацювання UI/UX та додавання важливого адміністративного функціоналу.

# План дій по доопрацюванню дизайну

## 1. Сторінка «AI-каталог»
- **Оптимізувати верхній блок:** перенести елементи керування (вкладки «AI-каталог» / «Акаунт-менеджмент», пошук, фільтр, експорт) в один верхній рядок для економії вертикального простору.
- **Змінити поле пошуку:** замінити текстове поле на іконку-кнопку (квадратик з лупою).
- **Прибрати зайві елементи:** видалити синю лінію під хедером — вона вибивається зі стилю.
- **Доопрацювати хедер/банер:**
  - спробувати додати новий банер із маскотами п’яти департаментів;
  - якщо банер виглядатиме недоречно, повністю прибрати верхній хедер.
- **Виправити іконку:** переробити іконку, що відображається некоректно (згадка від Елеонори).

## 2. Відображення статусу інструментів (на обох сторінках)
- **Замінити текст на іконки:**
  - для платних (paid) та умовно-платних (freemium) додати іконку корони у правому верхньому куті картки;
  - для повністю безкоштовних (free) іконку не додавати.
- **Позначення доступу через акаунт:** продумати візуальний індикатор для інструментів на кшталт GitHub, що безкоштовні, але потребують входу з корпоративного акаунта.

## 3. Сторінка «Акаунт-менеджмент»
- **Змінити структуру:** замість горизонтального скролу з категоріями (free, paid) зробити дві вертикальні колонки — для безкоштовних і для платних інструментів.
- **Спростити картки:** розглянути варіант карток акаунтів лише з логотипами інструментів без назв.
- **Очистити дані:** видалити неіснуючу пошту «IT-саппорт» зі списку акаунтів.

## 4. Реалізація CRUD-функціоналу (Create, Read, Update, Delete)
- **Додати «режим редагування»:**
  - загальна кнопка-перемикач (іконка олівця) поруч із перемикачем темної теми;
  - при натисканні інтерфейс переходить у режим редагування/додавання елементів.
- **CRUD для карток інструментів:**
  - створення нових карток (через поп-ап/модальне вікно);
  - редагування існуючих (клік по картці в режимі редагування);
  - видалення карток.
- **CRUD для акаунтів і тегів:** за аналогією з картками інструментів забезпечити додавання, редагування та видалення акаунтів і тегів.
